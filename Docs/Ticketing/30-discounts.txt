Требование:	
	предоставить пользователю возможность добавлять скидки определенной группе гостей.

Сценарии:
	BACKGROUND:
		GIVEN Пользователь находится на странице создания/редактирования мероприятия
		  AND Тип мероприятия - ticketing
		  AND Вкладка Discount активна.
		  
ОБЩИЕ:
	1. GIVEN Нет созданных скидок.
	    WHEN Пользователь кликает кнопку "NEXT".
	    THEN Пользователь переходит на следующую вкладку.	
		
ДОБАВЛЕНИЕ:
	2.  WHEN Пользователь кликает кнопку "ADD CODES".
	    THEN Пользователь видит новую форму для создания скидки с кнопками "OK" и "CANCEL"
		 AND Поле размера скидки заполнено значением 0, которое является минимально возможным
		 AND Максимально возможное значение равно цене самого дешевого типа билетов
		 AND Кнопки "ADD" и "CREATE", служащие для добавления гостей, не активны.
		 
	3. GIVEN Пользователь вводит допустимое значение в поле размера скидки.
	    WHEN Пользователь нажимает кнопку "OK".
	    THEN На сервере создается новая скидка 
		 AND Кнопка "DELETE" появляется
		 AND Кнопки "ADD" и "CREATE" становятся активными.
		 
	4. GIVEN Пользователь добавил новую скидку.
	    WHEN Пользователь нажимает кнопку "CANCEL".
	    THEN Форма для создания скидки исчезает.
	
	5. GIVEN Пользователь сохранил скидку нажав "OK".
	    WHEN Пользователь нажимает кнопку "ADD" и "CREATE".
	    THEN Пользователь проходит все шаги добавления/создания списков гостей, которые реализованы для других типов мероприятий
		 AND Всем пользователям из списка назначается DISCOUNT CODE
		 AND Добавленный список является приватным.

	6. GIVEN Пользователь сохранил скидку нажав "OK".
	    WHEN Пользователь нажимает кнопку "ADD" и "CREATE".
	    THEN Пользователь проходит все шаги добавления/создания списков гостей, которые реализованы для других типов мероприятий
		 AND Всем пользователям из списка назначается DISCOUNT CODE.
		 
	7. GIVEN Пользователь нажал кнопку "ADD CODES" 3 раза
	     AND Появились 3 новые несохраненные скидкис кнопками "OK" и "CANCEL"
		 AND Пользователь сохранил один из них нажатием на кнопку "OK".
	    WHEN Пользователь нажимает кнопку "NEXT".		 
	    THEN Пользователь переходит на след. секцию
		 AND 2 не сохраненных скидки удаляются.
		 

РЕДАКТИРОВАНИЕ:

	9.   WHEN Пользователь изменяет значение существующей скидки.
	     THEN Появляются кнопки "OK" и "CANCEL"
		  AND Исчезает кнопка "DELETE".
			 
УДАЛЕНИЕ:

	10. GIVEN Мероприятие не опубликовано
		 AND Проданных билетов нет.
	    WHEN Пользователь выбирает одну из сохраненных скидок
		 AND Нажимает кнопку "DELETE"
		 AND Подтверждает удаление.
	    THEN Скидка удаляется на сервере
		 AND Все DISCOUNT CODES для списков гостей удаляются.
		 
	11. GIVEN Мероприятие опубликовано
		  AND Продано 2 билета со скидками.
	     WHEN Пользователь выбирает одну из сохраненных скидок
		  AND Нажимает кнопку "DELETE"
		  AND Подтверждает удаление.
	     THEN Пользователь получает сообщение "You cannot delete this discount, there are bought tickets"
		  AND Скидка остается не удаленной.		

Server side:
	API:
		1. GetDicscounts/EventId - GET
		2. AddDicscount          - POST
		3. BindListToDicscount   - POST
		4. UpdateDiscount        - POST  
		5. DeleteDiscount        - POST
